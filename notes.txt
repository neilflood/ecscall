- Module should export a single function, something like callFunc().
- Internally there will be a class which will look rather like
  RIOS's ECSComputeManager
- Also a class equivalent to RIOS's NetworkDataChannel. It should have
  the following items
  - a queue of call argument tuples. Each object on the queue has
    a list index (i.e. where it came from in the original list),
    and a tuple of arguments with which to call the function
  - the function which will be called. This is the same function
    for all calls. (Use cloudpickle to transfer it)
  - a dictionary into which output objects are put(). Key is the
    list index, value is the function return value for that index.
  - a forceExit event
  - an exception queue, where worker exceptions are placed
  - a worker barrier
- The main call should take all the parameters to define how the workers
  will be set up, including things like the container to use, the various
  parameters like subnet, roles, etc. Should this be a dictionary
  very similar to how RIOS does it? That would allow for maximum 
  flexibility. Also allows it to be a single argument, constructed
  beforehand, which would be more robust against changes.
- Number of workers should be an explicit parameter. Question about
  whether it should be optional - this may allow for future options of
  instead specifying a max runtime, allowing dynamic addition of workers??
